# Homepage vs /product-intro 内容对比分析

> 日期：2026-03-30

---

## 一、内容结构对比

| 维度 | Homepage (Landing.tsx) | /product-intro (ProductIntro.tsx) |
|---|---|---|
| **语言** | 英文为主 | 中文为主（8语言 i18n） |
| **目标受众** | 偏技术/开发者（强调API、ATS集成） | 偏业务决策者/HR（强调痛点、价值） |
| **核心叙事** | 产品功能展示 | 痛点→方案→价值→对比→场景 |
| **Hero** | "AI Screens. AI Interviews. You Hire the Best." | "从需求到录用，全流程 AI 自动化" |
| **流程展示** | 4步（偏简略） | 6步（详细，含每步AI Agent说明） |
| **对比表** | 无 | 10维度传统 vs RoboHire对比 |
| **痛点** | 无专门板块 | 7个痛点深度描述（含初创企业） |
| **差异化** | 无专门板块 | 4个关键差异 |
| **真实场景** | 无 | "周一到周三"完整叙事 |
| **客户证言** | 4条（虚构英文名） | 1条（真实校招场景） |
| **目标客户** | 无 | 5类细分客户 |
| **定价** | 无 | 4档RMB定价 |
| **FAQ** | 7个问答 | 无 |
| **主题切换** | 无 | 有Light/Dark toggle |
| **视觉设计** | 标准landing page | 深蓝渐变+玻璃拟态+网格背景 |

## 二、各自优势

### Homepage 的优势
- 有FAQ板块（SEO友好）
- 有开发者API入口（技术用户转化）
- 有3种服务卡片（Start Hiring / Quick Invite / API）
- 有ATS集成展示（Greenhouse, Lever等）
- SEO Schema标记（HowTo, FAQPage, AggregateRating）

### ProductIntro 的优势
- **叙事逻辑更强**：痛点→方案→对比→场景→证言→定价，完整的销售漏斗
- **情感共鸣**：7个痛点直击HR日常困境
- **数据说服力**：10维度对比表一目了然
- **场景化**：周一到周三的真实场景让客户能代入
- **覆盖初创企业**：专门针对"无HR团队"的痛点和价值主张
- **有定价**：降低决策摩擦
- **视觉更现代**：深蓝主题+玻璃拟态设计更有品质感
- **中文本地化**：对中国市场更有说服力

## 三、结论

**`/product-intro` 更适合作为 Homepage**

### 原因
1. **转化逻辑完整** — 经典的 Problem → Solution → Proof → CTA 销售叙事
2. **情感驱动** — 痛点描述让访客立刻产生共鸣
3. **差异化清晰** — 传统 vs RoboHire 对比表是最有力的说服工具
4. **场景具象** — "42天→3天" 的故事比抽象数据更有说服力
5. **覆盖更广** — 从初创到大企业全覆盖

### 整合到首页时需要补充
- 从Homepage移植 **FAQ 板块**（SEO价值高）
- 补充 **开发者/API 入口**（不丢失技术用户）
- 添加 **ATS集成展示**（增强企业信任感）
- 增加更多 **客户证言**（1条不够，至少3-4条）
- 补充 **SEO Schema 标记**（HowTo, FAQPage, AggregateRating）
- 添加 **GEO 元数据**（地理位置定向）
