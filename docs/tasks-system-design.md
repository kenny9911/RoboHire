# Tasks System — Design Document

> **Version**: 1.0  
> **Date**: 2026-04-09  
> **Status**: Implementation  

---

## 1. Overview

The Tasks system transforms RoboHire from a tool-driven platform into an **action-driven command center**. Every significant recruitment event automatically generates a task — either for an AI agent to execute autonomously or for a human recruiter to make a judgment call.

### Goals
- **Zero missed actions**: Every interview completion, high match, stale candidate, or new job triggers an appropriate follow-up
- **Human/Agent separation**: AI handles data processing tasks automatically; humans handle judgment calls with full context
- **SLA enforcement**: Configurable time-based deadlines with escalation chains
- **Productivity tracking**: Task completion metrics feed into recruiter KPIs
- **Flexibility**: Admins can enable/disable task types, adjust SLA thresholds, and configure auto-execution

## 3. Data Models

### 3.1 Task

```prisma
model Task {
  id            String    @id @default(cuid())
  userId        String    // assignee
  createdById   String?   // who/what created it (null = system)

  // Classification
  type          String    // task type enum (see §4)
  category      String    // pipeline, evaluation, sourcing, communication, admin
  assigneeType  String    @default("human") // human | agent

  // Content
  title         String
  description   String?   @db.Text
  actionUrl     String?   // deep link to relevant page
  actionLabel   String?   // "Review Evaluation", "Shortlist Candidate"

  // Context links
  jobId              String?
  resumeId           String?
  interviewId        String?
  candidateId        String?
  matchingSessionId  String?
  agentId            String?
  hiringRequestId    String?

  // Priority & SLA
  priority      String    @default("medium") // critical, high, medium, low
  dueAt         DateTime?
  slaDeadline   DateTime?
  escalatedAt   DateTime?

  // Status
  status        String    @default("pending") // pending, in_progress, completed, dismissed, expired, auto_completed
  completedAt   DateTime?
  completedBy   String?   // userId or 'system'
  dismissReason String?
  result        Json?     // outcome data (e.g., evaluation score, match count)

  // Trigger
  triggerEvent  String    // what event spawned this
  triggerData   Json?     // event context snapshot
  isAutoGenerated Boolean @default(true)

  // Archive
  archivedAt    DateTime? // set 7 days after completed/dismissed

  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  // Relations
  user          User      @relation("TaskAssignee", fields: [userId], references: [id], onDelete: Cascade)
  createdBy     User?     @relation("TaskCreator", fields: [createdById], references: [id], onDelete: SetNull)
  job           Job?      @relation(fields: [jobId], references: [id], onDelete: SetNull)
  resume        Resume?   @relation(fields: [resumeId], references: [id], onDelete: SetNull)
  interview     Interview? @relation(fields: [interviewId], references: [id], onDelete: SetNull)

  @@index([userId, status])
  @@index([userId, priority])
  @@index([type, status])
  @@index([status, dueAt])
  @@index([jobId])
  @@index([interviewId])
  @@index([archivedAt])
}
```

### 3.2 TaskAutomationRule

```prisma
model TaskAutomationRule {
  id              String    @id @default(cuid())
  userId          String?   // null = global default; set = per-org override
  taskType        String    @unique // maps to task type enum
  enabled         Boolean   @default(true)
  assigneeType    String    @default("human") // human | agent
  autoExecute     Boolean   @default(false) // for agent tasks: execute immediately?
  slaHours        Int?      // SLA deadline in hours
  priority        String    @default("medium") // default priority
  escalateAfterHours Int?   // hours after SLA breach to escalate
  emailNotify     Boolean   @default(false) // send email on creation?
  config          Json?     // extra type-specific config
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

  @@index([taskType])
  @@index([userId])
}
```

### 3.3 Notification

```prisma
model Notification {
  id          String    @id @default(cuid())
  userId      String    // recipient
  taskId      String?   // linked task
  type        String    // task_created, task_overdue, task_escalated, task_completed
  title       String
  message     String?
  actionUrl   String?
  read        Boolean   @default(false)
  readAt      DateTime?
  emailSent   Boolean   @default(false)
  createdAt   DateTime  @default(now())

  user        User      @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId, read])
  @@index([userId, createdAt])
}
```

## 5. Priority Scoring Engine

Priority is auto-calculated but can be manually overridden.

### Composite Score (when auto-calculating):

```
score = (stageWeight * 0.30)
      + (slaProximity * 0.25)
      + (matchQuality * 0.20)
      + (roleCriticality * 0.15)
      + (staleness * 0.10)
```

| Factor | Weight | Scoring |
|--------|--------|---------|
| Pipeline stage depth | 30% | offer=100, interview=80, screening=60, matched=40, sourced=20 |
| SLA proximity | 25% | >100% overdue=100, >75%=80, >50%=50, <50%=20 |
| Match quality | 20% | A+=100, A=80, B+=60, B=40, C=20, D/F=0 |
| Role criticality | 15% | headcount>3=80, headcount>1=50, headcount=1=30 |
| Staleness | 10% | >14d=100, >7d=70, >3d=40, <3d=10 |

**Mapping**: score ≥ 80 → critical, ≥ 60 → high, ≥ 30 → medium, < 30 → low

## 6. SLA & Escalation Engine

### Lifecycle:

```
Task Created (T=0)
    │
    ├─── T < 75% SLA ───── Normal display
    │
    ├─── T = 75% SLA ───── Warning indicator (amber)
    │
    ├─── T = 100% SLA ──── OVERDUE: priority auto-escalates one level
    │                       Notification created (task_overdue)
    │
    └─── T = 150% SLA ──── URGENT badge, pinned to top of inbox
                            Email notification sent
                            escalatedAt timestamp set
```

### Auto-archive:
- Completed/dismissed tasks: `archivedAt = completedAt + 7 days`
- A scheduled job runs daily to set `archivedAt` on eligible tasks
- Archived tasks are excluded from default queries but remain queryable

```
          │        │          │        │
          │        └──────────┘        │

     [dismiss]      [start]       [auto-exec]
          │              │             │
          ▼              ▼             ▼
   ┌──────────┐   ┌──────────┐  ┌──────────────┐
   │dismissed │   │in_progress│  │auto_completed│
   └──────────┘   └──────────┘  └──────────────┘
                       │
                  [complete]
                       │
                       ▼
                 ┌──────────┐
                 │completed │
                 └──────────┘

Expired: pending tasks past SLA deadline with no action → status=expired
```
